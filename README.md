Sobi AutoParts

Project Description
Sobi AutoParts is a full-stack web-based marketplace platform where users can buy and sell auto parts. It supports fixed-price sales, auctions, and bargaining (price negotiation).
The system is built as part of a team-based software engineering course using the MERN stack and follows Hexagonal Architecture in the backend.

Purpose
The system allows:
Sellers to list auto parts using different selling methods
Buyers to browse, bid, negotiate prices, and purchase products
Admins to manage users and listings
The system does not include physical delivery or payment processing.

User Roles

Buyer
Browse and search products
Place bids and offers
Purchase products
View purchase history
Seller
Create and manage listings
Set fixed price, auction, or bargaining options
Manage bids and offers
Admin
Manage users and listings
Monitor platform activity
Features
User registration and authentication
Role-based access control
Product listing management
Auction system with bidding
Bargaining system with offer management
Product search and filtering
Purchase tracking

Architecture

Backend follows Hexagonal Architecture:
Domain Layer: Core business logic and entities
Application Layer: Use cases and services
Ports: Interfaces for communication
Adapters: Database and API implementations

Setup Instructions
Clone repository:
git clone https://github.com/bhatti-zayan/Sobi-AutoParts.git

Frontend:
cd frontend

npm run dev

Backend:
cd backend
npm install
npm run dev
